Anti-Money Laundering Detection System

This project applies network analysis and distributed computing to financial transaction monitoring‚Äîcombining graph-based fraud detection with real-time streaming data processing.

Current Implementation

I've built a baseline rule-based detection system that serves two purposes: providing immediate functionality for the dashboard and establishing performance benchmarks for the graph neural network (GNN) currently in development. The dashboard displays real-time results from this system as transactions are processed.

Architecture

Transaction streams flow through Azure Event Hubs into Databricks, where each transaction is evaluated against the existing transaction graph stored in Azure Blob Storage. Detection results are written to Azure Cosmos DB, and the dashboard updates via Azure Functions API calls that query the most recent processed transactions.

Azure Event Hubs Azure Blob Storage Azure Cosmos DB Azure Functions Databricks PySpark Python GitHub Pages

Note on Detection Rates

Due to the naturally low frequency of money laundering patterns and the conservative nature of the rule-based approach, periods showing zero detections are expected during normal operation. This will improve as the GNN model is integrated.
